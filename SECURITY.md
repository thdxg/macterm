# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| Latest  | Yes       |

## Reporting a Vulnerability

If you discover a security vulnerability in Macterm, please report it responsibly.

**Do not open a public issue for security vulnerabilities.**

Instead, please use [GitHub's private vulnerability reporting](https://github.com/macterm-app/macterm/security/advisories/new) to submit your report.

### What to include

- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### What to expect

- Acknowledgment within 48 hours
- A plan for a fix within 7 days for confirmed issues
- Credit in the release notes (unless you prefer to remain anonymous)

## Scope

This policy applies to the Macterm application and its build tooling. For vulnerabilities in [libghostty](https://github.com/ghostty-org/ghostty), please report them to the Ghostty project directly.
