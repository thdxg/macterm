# Macterm website

The marketing landing page and docs, served by a Cloudflare Worker
(`src/index.js`) that also exposes a `/api/stats` endpoint for live GitHub
stars and the latest `.dmg` download link.

## Structure

```
public/            Served as static assets by the Worker
  index.html       Landing page (hand-authored)
  docs.html        Docs page  ── generated, do not edit ──
  tailwind.css     Compiled styles ── generated, do not edit ──
  site.js          Shared behavior (sticky nav, copy, scroll-spy, stats)
  assets/          icon.png, screenshot.png, JSON schemas
src/
  tailwind.css     Tailwind v4 input + design tokens (@theme) + components
  docs-template.html  Shell the rendered docs are injected into
docs/
  docs.md          Docs content, authored in Markdown
build-docs.mjs     Renders docs/docs.md → public/docs.html
```

## Develop

```sh
bun install
bun run build       # build:docs then build:css
bun run dev         # builds, then wrangler dev
bun run watch:css   # rebuild CSS on change (run build:docs by hand for content)
```

`docs.html` and `tailwind.css` are build artifacts (gitignored) — regenerated by
`bun run build`, which runs automatically before `dev` and `deploy`. Edit the
docs by changing `docs/docs.md`; edit styles/tokens in `src/tailwind.css`.

Docs section markers in `docs.md` drive both the page sections and the sidebar:

```
<!-- section: <id> | <Nav label> | <Group name> -->
```

Fenced code blocks render as the dark code component; add `title="path"` after
the language for a filename caption bar.

## Deploy

```sh
bun run deploy      # build + wrangler deploy
```
